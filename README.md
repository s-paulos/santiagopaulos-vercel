# Santiago Paulós — Vercel Site

Static site recreation of santiagopaulos.com, ready for free Vercel hosting.

---

## Project structure

```
santiagopaulos-vercel/
├── index.html                  ← source (WP image URLs)
├── vercel.json                 ← Vercel config
├── scripts/
│   ├── download-images.js      ← run ONCE to pull all images locally
│   └── build.js                ← rewrites index.html with local /images/ paths
└── public/                     ← Vercel serves this folder
    ├── formspree-config.json   ← contact form endpoint (email hidden from HTML)
    ├── image-map.json          ← generated by download-images.js
    └── images/                 ← generated by download-images.js (~180 images)
```

---

## Step-by-step: deploying to Vercel

### 1. Prerequisites
- [Node.js](https://nodejs.org) installed (any version ≥ 14)
- A [Vercel account](https://vercel.com) (free)
- A [GitHub account](https://github.com) (free, easiest way to deploy)

### 2. Download all images from WordPress

Run this **before** cancelling your WordPress subscription:

```bash
cd santiagopaulos-vercel
node scripts/download-images.js
```

This downloads ~180 images into `public/images/` and creates `public/image-map.json`.
It takes 2–5 minutes. Images already downloaded are skipped on re-runs.

### 3. Build the final HTML with local image paths

```bash
node scripts/build.js
```

This writes `public/index.html` with all WordPress CDN URLs replaced by `/images/…` paths.

### 4. Verify locally (optional)

```bash
npx serve public
# open http://localhost:3000
```

### 5. Deploy to Vercel

**Option A — GitHub (recommended):**
1. Create a new GitHub repo (public or private)
2. Push this entire folder to it
3. Go to [vercel.com](https://vercel.com) → Add New Project → Import your repo
4. Vercel auto-detects `vercel.json` and runs `node scripts/build.js`
5. Done — your site is live at `your-project.vercel.app`

**Option B — Vercel CLI:**
```bash
npm i -g vercel
vercel --prod
```

### 6. Custom domain (optional)

In your Vercel dashboard → Project → Settings → Domains, add `santiagopaulos.com`.
Then update your domain's DNS to point to Vercel (they show exact instructions).

## Contact form

The form uses [Formspree](https://formspree.io) to deliver messages to your inbox.

- Your email address is **not in the HTML source** — it's loaded from `public/formspree-config.json` at runtime
- Formspree's free tier allows 50 submissions/month (plenty for an artist site)
- To activate: log in to formspree.io with spaulos@gmail.com and verify the form

**Note on the endpoint:** The endpoint in `formspree-config.json` uses your email address as the form ID. Formspree supports this format, but if you'd prefer a completely opaque ID (e.g. `https://formspree.io/f/abcde123`), create a form on the Formspree dashboard and replace the value in that file.

## Cancelling WordPress

Only cancel WordPress **after**:
- ✅ `node scripts/download-images.js` completed successfully
- ✅ `node scripts/build.js` completed successfully
- ✅ Site is live and all images load on Vercel

---

## Updating the site

To add new work in the future, edit `index.html` — specifically the `W` object in the `<script>` block. Add your image to `public/images/`, update the data, and re-deploy (push to GitHub and Vercel auto-deploys).
